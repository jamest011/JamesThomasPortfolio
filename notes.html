    • Single-Page Apps

  • What? Website that loads a single web page then loads it with JS.
  • Provides better loads and a more dynamic experience for the user, but you have tradeoffs.
  • More effort to maintain, monitor the performance and limit SEO.
  • Route matching compares the current location against the routes to find matches.
  • history.pushState() - adds an entry to the history after the current one can update the URL without triggering a page reload
  • history.replaceState() - replaces current entry.
  • go() - same as forward button on browser. Ex: go(-1) = go back one, go(1) = go forward one, go() = reload
  • using history.go() and clicking forward/back buttons to navigate will cause full page reloads.
  • More Info - https://developer.mozilla.org/en-US/docs/Web/API/History_API
  • At times 'You end up recreating with JavaScript a lot of the features the browser gives you out-of-the-box.'
  • Doing this means that you have more code to maintain and can make the code more bug-prone.
  • handle routing - https://krasimirtsonev.com/blog/article/A-modern-JavaScript-router-in-100-lines-history-api-pushState-hash-url
  • A simpler approach is to let the browser load the real HTML files located at the URLs.
  • Why is this better than JS routing?
  • Supports built in browser functions for forward and backwards.
  • No need to intercept clicks
  • No complex regex patterns or using another library to parse the URL. It's built in the markup
  • Simpler and easier to implement.

    • Getting Started with JSON Web Tokens

  • JSON Web Token (JWT) - open standard that defined a compact and self-contained way for securely transmitting info between parties as a JSON object.
  • Compact - can be sent through URL, POST param or in an HTTP header and is fast.
  • Self-contained - contains all the required info about the user to avoid querying database more than once
  • JWTs are a good way to securely transmit info between parties because you can be sure that the user is who they say they are.
  • JWTs consist of 3 parts: Header, Payload and Signature
  • DO NOT STORE SENSITIVE DATA IN BROWSER STORAGE DUE TO LACK OF SECURITY
  • Why use JWTs
  • smaller in size
  • easy client-side processing on multiple platforms especially mobile
